# Target Research Skill

## What to Find (in this order)

### 1. Company Basics
- Industry and sub-sector
- Headcount and revenue (Companies House for UK)
- Locations — HQ and satellite offices
- Recent funding rounds or M&A activity

### 2. Tech Stack Signals
- Job listings: DevOps, Cloud, Infrastructure roles reveal stack
- LinkedIn: engineer profiles often list AWS/Azure/VMware
- G2 / Crunchbase: technology tags
- Their website: "built on AWS" badges, case studies

### 3. Buying Triggers (look for these)
| Signal | Means | Hook |
|--------|-------|------|
| VMware in job listings | On-prem VMware — Broadcom pain | VMware |
| "AWS cost" in Glassdoor reviews | Overspending | Cost Savings |
| Hiring Cloud Architect | Building cloud team, need help | Any |
| Recent funding round | New budget, need to scale fast | Cost or AI |
| Security job listings | Security gap, risk-aware | Security |
| "AI strategy" in comms | AI ambition, need delivery | Agent Bakery |
| Contract renewal (VMware/DC) | Forced migration moment | VMware |

### 4. Decision Maker
- Target: CTO, VP Engineering, IT Director, or CEO (for SMB)
- LinkedIn for current title and tenure
- Check if they follow AWS accounts — signal of AWS interest

### 5. Conversation Starter
One specific thing to reference in the subject line or opening:
- A recent news story about them
- A specific job listing
- An award or recognition
- A product launch

## Output Format
Produce for each company:
1. Company snapshot (5 bullets)
2. Top buying trigger identified
3. Best hook to use
4. Target persona (title + name if findable)
5. Conversation starter
6. Confidence level: HIGH / MEDIUM / LOW
