# Customer Onboarding Skill

## The 30-Day Programme

| Days | Milestone | Owner |
|------|-----------|-------|
| D1–3 | Kickoff call, access confirmed, tooling set up | Delivery lead |
| D7 | Environment assessment complete, first findings shared | Technical |
| D14 | Quick wins implemented (Tier 1 FinOps, security basics) | Technical |
| D21 | **First Win Delivered** — specific £ saving or risk closed | Delivery lead |
| D25 | First Value Report drafted | Delivery lead |
| D30 | **First Value Report delivered** — 30-day review call | Steve / Oliver |

## First Win (by Day 21)
Must be specific and measurable. Examples:
- "Identified and removed £800/month of idle resources"
- "Enabled GuardDuty — found and closed 3 critical findings"
- "Rightsized 4 EC2 instances — saving £400/month"

Never let Day 21 pass without a documented win. If blocked by access issues, escalate immediately.

## First Value Report (Day 30)
One page. Covers:
1. What we set up
2. What we found
3. What we fixed (with £ value where possible)
4. What we're doing next month
5. One chart: AWS spend before/during onboarding

## Onboarding Health Check
Green flags: access granted, team engaged, first win delivered
Amber flags: slow access, low engagement, no first win by D21
Red flags: no access by D7, sponsor not engaged, scope creep requests
